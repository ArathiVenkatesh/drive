✅ ✅ EXPERIMENT 13
Integrating JUnit with Jenkins (GitHub CI)
🔹 PART 1 – Maven Project Creation & Local Testing

mvn -version
mvn archetype:generate -DgroupId=com.bnmit -DartifactId=sample-app -DarchetypeArtifactId=maven-archetype-quickstart -DinteractiveMode=false
cd sample-app
mvn clean test
Expected Output:

Tests run: 1
Failures: 0
BUILD SUCCESS
🔹 PART 2 – Push Project to GitHub

git init
git add .
git commit -m "Initial commit - Maven JUnit project"
git branch -M main
git remote add origin https://github.com/USERNAME/sample-app.git
git push -u origin main
🔹 PART 3 – Jenkins Maven Execution
In Jenkins (Maven Project Job), under Goals and Options, enter:

package
Internally Jenkins runs:

mvn package
Which automatically performs:
compile
test
package
🔹 What Jenkins Executes During Build
Behind the scenes:

mvn clean package
JUnit tests are executed automatically.
🔹 Expected Jenkins Console Output

Running com.bnmit.GreatestNumberTest
Tests run: 1, Failures: 0, Errors: 0, Skipped: 0
BUILD SUCCESS
Artifacts generated:

target/sample-app-1.0-SNAPSHOT.jar
✅ ✅ EXPERIMENT 14
Measuring Code Coverage Using JaCoCo
🔹 PART 1 – Create Maven Project

mvn -version
mvn archetype:generate -DgroupId=jar -DartifactId=Lab14-JaCoCo -DarchetypeArtifactId=maven-archetype-quickstart -DinteractiveMode=false
cd Lab14-JaCoCo
🔹 PART 2 – Run Tests with JaCoCo
After adding JaCoCo plugin in pom.xml:

mvn clean test
This automatically:
Runs JUnit tests
Collects coverage data
Generates report
🔹 Generated Files
Inside target/ folder:

target/jacoco.exec
target/site/jacoco/index.html
🔹 Open Coverage Report

target/site/jacoco/index.html
